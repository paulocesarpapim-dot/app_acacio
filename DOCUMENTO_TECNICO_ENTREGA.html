1. Arquitetura Geral do Sistema

O site Empório Filho de Deus é uma aplicação web moderna do tipo SPA (Single Page Application) com uma API REST serverless. A arquitetura é dividida em três camadas:

Como funciona o deploy

O deploy é automático. Sempre que um git push é feito para o branch main no GitHub, a Vercel detecta as mudanças e executa automaticamente:

  1. Instala as dependências (npm install)
  2. Compila o frontend (npm run build → gera a pasta dist/)
  3. Empacota as Serverless Functions (api/handler.js)
  4. Distribui globalmente pela CDN da Vercel
  5. Ativa o novo deploy em produção

Como os dados são armazenados

Todos os dados do sistema são armazenados em uma única chave Redis chamada database, que contém um JSON com a seguinte estrutura:

{
  "products": [...],      // Lista de produtos
  "customers": [...],     // Lista de clientes cadastrados
  "orders": [...],        // Pedidos realizados
  "settings": {...},      // Configurações (Pix, categorias, senhas)
  "promotions": [...]     // Promoções ativas
}

Fluxo de persistência

  1. A cada escrita (produto, cliente, pedido), o sistema salva no arquivo /tmp/database.json e simultaneamente no Redis
  2. Em um cold start (primeira requisição após inatividade), o sistema lê do Redis e restaura os dados
  3. Produtos iniciais vêm do arquivo database.json do deploy, mas depois de atualizados, a versão do Redis tem prioridade
O plano gratuito do Upstash permite 10.000 comandos/dia. Para um e-commerce com tráfego moderado, isso é suficiente. Se o limite for atingido, considere migrar para o plano Pay-As-You-Go ($0.2/100K comandos).
Para monitorar o uso, acesse o painel: https://console.upstash.com/redis/ff06ab0f-2dcf-40fb-b7ce-e827cc141279. A aba "Usage" mostra os comandos consumidos.

Alteração de Senha do Admin

A senha do painel administrativo pode ser alterada de duas formas:

Opção 1: Pela Vercel (recomendado)

  1. Acesse Vercel → casadonorte → Settings → Environment Variables
  2. Edite a variável ADMIN_PASSWORD com a nova senha
  3. Faça um redeploy para ativar

Opção 2: Pela API (senha já hashada no Redis)

Após o primeiro login com sucesso, a senha é automaticamente hasheada com scrypt e salva no Redis. Para resetar, atualize a variável ADMIN_PASSWORD na Vercel e limpe o campo settings.adminPassword no Redis.

4.1 Gerenciar Produtos

Adicionar um produto

No painel admin, use o formulário de cadastro com os campos:

CampoObrigatórioDescrição
Nome✅ SimNome do produto (ex: "Feijão Preto")
DescriçãoNãoDescrição detalhada
Categoria✅ SimDeve coincidir com uma categoria existente
Preço✅ SimValor em reais (ex: 6.99)
URL da ImagemNãoURL de uma imagem. Se vazio, usa placeholder

Editar um produto

Clique no botão de editar ao lado do produto, altere os campos desejados e salve. As alterações são salvas no Redis e persistem entre deploys.

Excluir um produto

Clique no botão de excluir. A ação é irreversível.

Via API (avançado)

# Login admin (obter token)
curl -X POST https://emporiofilhodedeus.com.br/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"password":"casadonorte2026"}'

# Criar produto
curl -X POST https://emporiofilhodedeus.com.br/api/products \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer SEU_TOKEN" \
  -d '{"name":"Feijão Verde","category":"Feijão","price":8.50}'

# Atualizar produto (ID = 1)
curl -X PUT https://emporiofilhodedeus.com.br/api/products/1 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer SEU_TOKEN" \
  -d '{"price":7.99}'

# Excluir produto (ID = 1)
curl -X DELETE https://emporiofilhodedeus.com.br/api/products/1 \
  -H "Authorization: Bearer SEU_TOKEN"

6. Configuração de Pagamentos

Pix (Estático — funciona sem API bancária)

O sistema gera cobranças Pix usando BRCode EMV padrão BACEN. As configurações padrão são:

ConfiguraçãoValor
Chave Pix64330427000130 (CNPJ)
NomeEMPORIO FILHO DE DEUS
CidadeSAO PAULO
BancoC6 Bank

Estas configurações podem ser alteradas no painel admin em Configurações.

Mercado Pago (Checkout Pro)

Para ativar pagamentos com cartão de crédito/débito via Mercado Pago:

  1. Obtenha o Access Token em https://www.mercadopago.com.br/developers
  2. Adicione como variável de ambiente na Vercel: MP_ACCESS_TOKEN
  3. Ou configure pelo painel admin em Configurações → Mercado Pago

Pix Dinâmico (C6 Bank API — Opcional)

Para cobranças Pix dinâmicas com confirmação automática via webhook:

  1. Configure as variáveis C6_CLIENT_ID e C6_CLIENT_SECRET na Vercel
  2. Registre o webhook via API: POST /api/pix/webhook/register

7. Segurança

MecanismoDescrição
Hash de SenhasTodas as senhas (clientes e admin) são hasheadas com scrypt (salt 16 bytes, key 64 bytes). Senhas em texto puro são migradas automaticamente no primeiro login.
Tokens de SessãoTokens admin são cryptographically random (32 bytes hex) com TTL de 8 horas.
Rate LimitingLogin admin e cliente limitado a 5 tentativas por 15 minutos por IP.
CORSAPI aceita requisições apenas de origens permitidas (localhost, .vercel.app, emporiofilhodedeus.com.br).
SSL/TLSCertificado HTTPS automático pela Vercel (Let's Encrypt).
Variáveis SensíveisTokens de pagamento e Redis são armazenados como variáveis de ambiente na Vercel, nunca no código.
Endpoint SeguroRotas administrativas protegidas por adminAuth middleware (Bearer token obrigatório).
Recomendações de segurança:
• Troque a senha do admin periodicamente
• Nunca inclua tokens ou senhas no código-fonte (use variáveis de ambiente)
• Monitore o painel da Upstash para detectar usos anormais
• Mantenha o repositório GitHub como privado

9. Resolução de Problemas

ProblemaCausa ProvávelSolução
Site fora do ar DNS ou Vercel Acesse /health. Se não responder, verifique a Vercel Dashboard e o DNS no Registro.br
API retorna 404 Deploy falhou Verifique na Vercel se o último deploy está como "Ready". Se necessário, force redeploy via git push
Produtos voltaram ao original Redis desconectado Verifique as variáveis UPSTASH_REDIS_REST_URL e UPSTASH_REDIS_REST_TOKEN na Vercel
"Muitas tentativas de login" Rate limiting Aguarde 15 minutos. O rate limit é por IP e reseta automaticamente
Senha admin não funciona Senha hasheada no Redis diferente da env var Atualize ADMIN_PASSWORD na Vercel, limpe settings.adminPassword no Redis e redeploy
Imagens não carregam URLs de imagem inválidas Atualize a image_url do produto com uma URL válida de imagem
Pix não gera QR Code Chave Pix inválida Verifique a chave Pix nas Configurações do admin
Erro 500 na API Erro interno Acesse Vercel → casadonorte → Deployments → Function Logs para ver o erro detalhado
